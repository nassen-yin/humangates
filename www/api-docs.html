概述

Human Gates 是 AI Agent 与物理世界之间的标准通信协议。通过统一的 REST API，开发者可以为 AI Agent 接入真人执行能力——公司注册、资质代办、银行面签、跑腿办事……只要是需要人跑一趟的事情，都能通过这一个 API 完成。

https://api.humangates.com/v1

当前已上线 公司注册，更多品类正在开发中。每个品类都有独立的结构化参数定义、内嵌的 validation 规则、以及完整的能力描述——AI Agent 可一次性读全、一次性填对、一次性通过审核。

设计理念

Human Gates 不只是一个工单系统。我们的核心价值在于：定义标准化的数据结构，让 AI 消除人与人之间的沟通成本。

注册公司最大的成本不是几百块工本费，而是资料来回修改、信息反复确认的时间。我们的 API 用深度结构化的 schema，强迫 AI Agent 第一次提交就把所有信息收集全——股东信息、章程条款、地址证明、高管任职……一个都不少。

v0.3 新增能力：

  • 能力描述（capabilities）：每个服务返回 agent_auto（10 项自动判定）、human_required（7 项人工事项）、escalation（5 个转人条件）
  • 名称驳回知识库：6 条全国通用规则，匹配成功自动生成替代方案
  • 经营范围映射表：20 组关键词到工商标准表述，全自动填充率约 80%
  • 城市分级：第一梯队 5 城（e签宝覆盖，零人工），第二梯队 7 城（待验证）
  • e签宝集成：5 个 API 端点，8 元/份，全程线上
  • 8 状态机：draft -> materials_collecting -> name_verification -> submitting -> in_progress -> needs_info -> approved -> completed

API 调用流程：

第一步：GET  /v1/services           → AI 读取可用服务 + 完整 JSON Schema + 能力描述 + 知识库
第二步：POST /v1/tasks               → 按 Schema 一次性提交结构化数据（初始状态: draft）
第三步：GET  /v1/tasks/{id}/progress → 实时查询进度百分比和各步骤状态
第四步：POST /v1/tasks/{id}/documents → 自动生成全套法律文书
第五步：等待外部服务回调            → e签宝完成实名认证→电子签名→工商递交
第六步：GET  /v1/tasks/{id}          → 获取最终结果

认证

API 请求通过 HTTP Header 传递密钥，区分开发者端和管理端。两张密钥独立配置，权限隔离。

Header用途适用端点
X-API-Key开发者密钥创建任务、查询任务、查看服务目录、进度查询、文书列表、审计日志
X-Admin-Key运营管理密钥更新状态、添加备注、文书生成、上传文件

查询任务列表

GET /v1/tasks 按状态过滤，支持分页
参数类型说明
statusstring?按状态过滤
limitnumber?每页数量（默认 20，最大 100）

更新任务状态

PUT /v1/tasks/{task_id}/status 运营端操作，受 8 状态机约束

v0.3 状态流转规则（8 状态）：

draft               ->  materials_collecting, needs_info
materials_collecting ->  name_verification, needs_info
name_verification    ->  submitting, needs_info
submitting           ->  in_progress, needs_info
in_progress          ->  needs_info, approved
needs_info           ->  materials_collecting, submitting
approved             ->  completed
completed            ->  (终态)
failed               ->  (终态)

工商局退回补件：

curl -X PUT https://api.humangates.com/v1/tasks/{task_id}/status \
  -H "X-Admin-Key: your-admin-key" \
  -H "Content-Type: application/json" \
  -d '{"status": "needs_info", "note": "经营范围表述不规范，请参照国民经济行业分类修改"}'

标记完成并回传结果：

curl -X PUT https://api.humangates.com/v1/tasks/{task_id}/status \
  -H "X-Admin-Key: your-admin-key" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "completed",
    "note": "执照已成功领取",

      "license_number": "91130100MAXXXXXX",
      "issue_date": "2026-05-08",
      "files": ["营业执照电子版已上传"]

文档生成（v0.3 新增）

POST /v1/tasks/{task_id}/documents 根据已提交数据自动生成法律文书

根据任务中已经提交的结构化数据，自动生成全套工商注册所需的法律文书草稿：

  • 公司章程 — 包含股东信息、出资条款、治理结构
  • 股东会决议 — 包含执行董事和监事任命
  • 法定代表人任职文件
  • 租赁合同（如 address_type=lease）

可选参数 doc_types 指定只生成部分文书。不传则生成全套。

curl -X POST https://api.humangates.com/v1/tasks/{task_id}/documents \
  -H "X-Admin-Key: your-admin-key" \
  -H "Content-Type: application/json" \
  -d '{"doc_types": ["articles_of_association", "shareholder_resolution"]}'

文件上传

POST /v1/tasks/{task_id}/files 上传身份证、合同等附件

支持格式：jpg / png / pdf / doc / docx。单文件上限 10MB。

curl -X POST https://api.humangates.com/v1/tasks/{task_id}/files \
  -H "X-Admin-Key: your-admin-key" \
  -F "file=@id_card_front.jpg" \
  -F "file_type=id_card_front" \
  -F "note=法人身份证正面"

file_type 取值：id_card_front / id_card_back / business_license / lease_contract / other

GET /v1/tasks/{task_id}/files 查看已上传文件列表 
curl https://api.humangates.com/v1/tasks/{task_id}/files \
  -H "X-API-Key: your-api-key"

状态含义
